INFORME TECNICO
Definicion de modulos y funcionalidades de la plataforma
Fecha: 01/04/2026

1) Objetivo de la plataforma
La plataforma de gestion de proyectos academicos permite registrar usuarios, crear proyectos,
hacer seguimiento del estado, almacenar revisiones, gestionar entregas, registrar evaluaciones
y manejar notificaciones. El backend esta construido en Node.js + Express y usa PostgreSQL
en Azure como base de datos principal.

3) Arquitectura general del backend
Flujo principal:
Cliente -> routes -> middleware -> controllers -> models -> PostgreSQL

Responsabilidad por capa:
- server.js: arranque del servidor y montaje global de rutas/middlewares.
- config/: conexion y acceso a base de datos.
- routes/: define endpoint + middlewares por endpoint.
- middleware/: autenticacion, autorizacion y manejo global de errores.
- controllers/: validaciones HTTP y orquestacion de logica.
- models/: consultas SQL parametrizadas.
- uploads/: almacenamiento de archivos (directorio preparado para entregas).

5) Funcionamiento por archivo (variables y funciones)

5.1 backend/server.js
Funcion general:
- Inicializa Express, conecta BD, monta middlewares y rutas, y levanta servidor.

Variables y su funcion:
- express: modulo para crear la app HTTP.
- cors: middleware para habilitar peticiones cruzadas.
- connectDB: funcion que intenta conexion inicial con PostgreSQL.
- pool: instancia compartida para consultas SQL (ej: /health).
- errorHandler: middleware global para errores no controlados.
- authRoutes: router de autenticacion bajo /api/auth.
- projectRoutes: router de proyectos bajo /api/projects.
- app: instancia principal de Express.
- PORT: puerto de escucha (process.env.PORT o 5001 por defecto).
- req, res (en endpoints): objetos de solicitud/respuesta HTTP.
- result (en /health): resultado de "SELECT NOW()" para validar BD.
- error (try/catch): captura de excepciones para responder 500.

Rutas activas en este archivo:
- /api/auth
- /api/projects
- /health
- /

5.2 backend/config/db.js
Funcion general:
- Configura la conexion a Azure PostgreSQL con Pool de pg.

Variables y su funcion:
- Pool: clase de pg para administrar conexiones reutilizables.
- path: utilitario para construir ruta segura al archivo .env.
- pool: pool de conexiones PostgreSQL usado por todo el backend.
- connectDB: funcion que prueba conexion al iniciar.
- process.env.PGHOST: host de Azure PostgreSQL.
- process.env.PGPORT: puerto de PostgreSQL (normalmente 5432).
- process.env.PGDATABASE: nombre de la base de datos.
- process.env.PGUSER: usuario de BD.
- process.env.PGPASSWORD: contrasena de BD.
- ssl.rejectUnauthorized: false para permitir certificado gestionado de Azure.

Resultado de exportacion:
- module.exports = { pool, connectDB }

5.3 backend/controllers/authController.js
Funcion general:
- Implementa registro y login de usuarios.

Variables y su funcion:
- bcrypt: libreria para hash/validacion de contrasenas.
- jwt: libreria para firmar/verificar tokens.
- User: modelo para operar tabla usuarios.
- req.body: datos enviados por cliente.
- nombre, email, password: campos de entrada.
- userExist: usuario encontrado por email para evitar duplicado.
- hashedPassword: contrasena encriptada con bcrypt.hash(password, 10).
- newUser: registro creado en BD.
- user: usuario encontrado al iniciar sesion.
- validPassword: resultado de bcrypt.compare(password, user.password).
- token: JWT firmado con payload { id, email, rol } y expiresIn "24h".
- process.env.JWT_SECRET: clave secreta para firmar token.
- error: captura de fallos y retorno de error 500.

5.4 backend/controllers/projectController.js
Funcion general:
- Implementa CRUD de proyectos y consulta de proyectos del usuario autenticado.

Variables y su funcion:
- Project: modelo SQL de proyectos.
- titulo, problema, justificacion, objetivos: campos requeridos para crear.
- estudiante_id: se obtiene de req.user.id (token autenticado).
- newProject: proyecto insertado en BD.
- projects: lista de proyectos (global o por estudiante).
- id: parametro de ruta para operaciones por proyecto.
- project: proyecto encontrado por id.
- updateData: objeto con campos a actualizar.
- updatedProject: resultado del UPDATE en BD.
- error: captura para respuestas de error.

5.5 backend/middleware/authMiddleware.js
Funcion general:
- Verifica autenticacion por JWT en rutas protegidas.

Variables y su funcion:
- jwt: libreria de verificacion de token.
- verifyToken: middleware principal.
- authHeader: valor del header Authorization.
- token: authHeader sin prefijo "Bearer ".
- decoded: payload decodificado del JWT (id, email, rol).
- req.user: se asigna con decoded para uso posterior.
- next: funcion para continuar al siguiente middleware/controlador.
- process.env.JWT_SECRET: clave para validar token.

5.6 backend/middleware/roleMiddleware.js
Funcion general:
- Autoriza acceso segun rol del usuario autenticado.

Variables y su funcion:
- checkRole(...rolesPermitidos): factory de middleware con roles permitidos.
- rolesPermitidos: arreglo de roles autorizados para una ruta.
- userRole: rol tomado de req.user?.rol.
- next: pasa control si el rol es valido.

Respuesta de control:
- 401 si no hay usuario autenticado.
- 403 si el rol no esta permitido.

5.7 backend/middleware/errorMiddleware.js
Funcion general:
- Formatea errores no controlados en una respuesta uniforme.

Variables y su funcion:
- err: error capturado por Express.
- req, res: objetos HTTP.
- next: parametro estandar de middleware de error.
- statusCode: err.statusCode o 500 por defecto.
- message: err.message o texto por defecto.

Formato de respuesta:
- { error: { status, message } }

5.8 backend/models/User.js
Funcion general:
- Operaciones SQL de usuarios.

Variables y su funcion:
- pool: ejecucion de consultas SQL.
- userData: objeto de entrada para crear usuario.
- nombre, email, password, rol: datos a insertar.
- query: sentencia SQL parametrizada.
- values: valores que reemplazan $1, $2, ...
- result: respuesta SQL con filas resultantes.
- id/email: filtros de busqueda.

Metodos:
- create, findByEmail, findById, findAll.

5.9 backend/models/Project.js
Funcion general:
- Operaciones SQL de proyectos.

Variables y su funcion:
- projectData: datos de proyecto para insercion.
- titulo, problema, justificacion, objetivos: campos de contenido.
- estudiante_id: FK al usuario creador.
- estado: estado del proyecto (default "propuesta" en modelo).
- id: identificador de proyecto.
- updateData: campos dinamicos para actualizar.
- fields: arreglo de asignaciones SQL (ej: "titulo = $1").
- values: arreglo de valores del UPDATE.
- paramIndex: contador para placeholders $n.
- result: resultado de consulta SQL.

Metodos:
- create, findById, findAll, findByEstudiante, update, delete.

5.10 backend/models/LineaTematica.js
Funcion general:
- Operaciones SQL de lineas tematicas.

Variables y su funcion:
- nombre: nombre de la linea a crear.
- id: identificador para busqueda.
- query, result: consulta y resultado SQL.

Metodos:
- create, findAll, findById.

5.11 backend/models/Entrega.js

Variables y su funcion:

- archivo: ruta o nombre de archivo entregado.

- result: resultado de consulta SQL.

5.13 backend/models/Notificacion.js
Funcion general:
- Operaciones SQL de notificaciones.

Variables y su funcion:
- usuarioData: datos para crear notificacion.
- usuario_id: destinatario de la notificacion.
- mensaje: contenido de la notificacion.
- id: identificador de notificacion.

Metodos:
- create, findByUsuario, markAsRead, findUnread.

5.14 backend/routes/authRoutes.js
Funcion general:
- Define rutas HTTP de autenticacion.

Variables y su funcion:
- express: framework para Router.
- router: instancia Router para agrupar endpoints.
- registerUser, loginUser: handlers importados de authController.

Rutas:
- router.post("/register", registerUser)
- router.post("/login", loginUser)

5.15 backend/routes/projectRoutes.js
Funcion general:
- Define rutas HTTP para CRUD de proyectos y aplica seguridad.

Variables y su funcion:
- router: instancia Router.
- projectController: handlers de proyectos.
- verifyToken: middleware de autenticacion JWT.
- checkRole: middleware de autorizacion por rol.

Rutas:
- GET / -> verifyToken -> getProjects
- GET /my-projects -> verifyToken -> getMyProjects
- GET /:id -> verifyToken -> getProjectById
- POST / -> verifyToken + checkRole("Estudiante") -> createProject
- PUT /:id -> verifyToken -> updateProject
- DELETE /:id -> verifyToken -> deleteProject

5.16 backend/testConnection.js
Funcion general:
- Valida conectividad a PostgreSQL desde terminal.

Variables y su funcion:
- pool: pool importado desde config/db.
- client: conexion tomada temporalmente del pool.
- result: salida de SELECT NOW().
- process.exit(0): termina exitosamente.
- process.exit(1): termina con error.

Uso:
- node testConnection.js
