# 📌 🔹 INTRODUCCIÓN E INSTRUCCIONES DEL PROYECTO (VERSIÓN INTEGRADA)

El presente proyecto tiene como objetivo desarrollar una **Plataforma Web Inteligente para la Gestión de Proyectos de Grado del ISER**, que permita registrar, organizar y hacer seguimiento a los trabajos de grado de manera digital, centralizada y eficiente.

Actualmente, la gestión de proyectos en instituciones educativas se realiza mediante procesos manuales, correos electrónicos y almacenamiento disperso de información, lo cual genera:

* Desorganización
* Falta de trazabilidad
* Retrasos en revisiones
* Dificultad en la toma de decisiones

Para solucionar esta problemática, se propone el desarrollo de una plataforma moderna basada en la integración y adaptación de los siguientes repositorios:

* [https://github.com/Cristhian2755/proyecto-grado.git](https://github.com/Cristhian2755/proyecto-grado.git)
* [https://github.com/elkynerojasiser/repositorio-iser.git](https://github.com/elkynerojasiser/repositorio-iser.git)

Estos servirán como base para construir un sistema robusto, escalable y orientado a roles institucionales.

---

# 🧩 🔹 ENFOQUE DE DESARROLLO

El sistema será desarrollado bajo una arquitectura web moderna basada en:

* API RESTful
* Separación Frontend / Backend
* Patrón MVC
* Control de acceso basado en roles (RBAC)

Esto permitirá una mayor escalabilidad, seguridad y mantenibilidad.

---

# 🚀 🔹 FUNCIONALIDADES DEL SISTEMA

La plataforma integrará las siguientes funcionalidades:

* 📚 Biblioteca pública de proyectos (sin autenticación)
* 🔐 Registro y autenticación con JWT
* 👥 Gestión de usuarios con roles
* 📄 Gestión completa del ciclo de vida del proyecto
* 📤 Subida y control de versiones de documentos
* 🔄 Flujo estructurado de aprobación
* 📝 Sistema de evaluaciones
* 🔔 Notificaciones automáticas
* 📊 Panel administrativo con métricas
* ⚠️ Detección de proyectos en riesgo (analítica básica)

# 🔄 🔹 FLUJO COMPLETO DEL PROYECTO

## 1. Creación del Proyecto (Estudiante)

El estudiante podrá:

* Registrar proyecto con:

  * Título
  * Introducción
  * Documento inicial (PDF)

El sistema automáticamente:

* Guarda estado: **PENDIENTE**
* Ejecuta módulo de IA para clasificación

---

## 2. Revisión del Docente

El docente podrá:

*lista de esstudiantes asignados
* Ver proyectos asignados
* Revisar contenido
* Realizar observaciones
* Cambiar estado a:

  * EN_CORRECCION
  * APROBADO_DOCENTE

---

## 3. Aprobación del Coordinador

El coordinador podrá:

* Registrar estudiantes (con programa académico)
* Asignar docentes
* Revisar proyectos aprobados
* Aprobar o rechazar
* Asignar jurados

Estados:

* APROBADO
* RECHAZADO

---

## 4. Clasificación Inteligente (IA)

Ubicado en: `/ia/`

El sistema permitirá:

* Analizar el contenido del proyecto
* Clasificar por área:

  * Redes y sistemas
  * Desarrollo de software
  * Seguridad informática
  * Otras (artes, salud, etc.)

📌 Resultado:

* Se guarda en la base de datos
* Sugiere posibles asesores

---

## 5. Seguimiento del Proyecto

Durante el desarrollo:

* El estudiante sube entregas (`Entrega.js`)
* El docente evalúa (`Evaluacion.js`)

El sistema genera:

* Notificaciones (`Notificacion.js`)
* Alertas de retraso
* Seguimiento continuo

---

## 6. Publicación en Biblioteca

Cuando el proyecto finaliza:

Se publica automáticamente en:

📁 `/docs/`

Visible para usuarios públicos.

Incluye:

* Título
* Autor
* Introducción
* Programa académico
* Documento final
