Dado que estás usando **Node.js** en el backend y tu base de datos centraliza roles (Biblioteca, Docentes/Coordinadores y Estudiantes), lo ideal es crear un **único servicio base** parametrizable o tres archivos independientes que configuren las **Instrucciones del Sistema (System Instructions)**. ## 1. El Controlador Base (`geminiChat.js`)

Para evitar duplicar código innecesario, crearemos una función que inicialice el chat inyectándole el **rol del sistema** correspondiente.

Crea un archivo de configuración o helper (por ejemplo en `backend/src/modules/AIClassificationModule/ia/geminiConfig.js` o dentro de una carpeta compartida de servicios si lo prefieres) o simplemente utilízalo como base para tus tres archivos.

Aquí tienes la implementación de los tres chatbots con sus respectivas instrucciones de sistema (*System Instructions*):

```javascript
import { GoogleGenerativeAI } from "@google/generative-ai";
import dotenv from "dotenv";

dotenv.config(); // Asegura leer el archivo .env de la raíz

const genAI = new GoogleGenerativeAI(process.env.GEMINI_API_KEY);

// Configuración compartida para los chats
const generationConfig = {
  maxOutputTokens: 800,
  temperature: 0.5, // Temperatura media-baja para respuestas precisas y menos creativas
};

// ==========================================
// 1. CHATBOT PARA BIBLIOTECA
// ==========================================
export const iniciarChatBiblioteca = () => {
  const model = genAI.getGenerativeModel({
    model: "gemini-1.5-flash",
    systemInstruction: `
      Actúas como el asistente virtual experto de la Biblioteca de la institución (iser_db). 
      Tu objetivo es ayudar al personal de biblioteca a gestionar, verificar y catalogar los proyectos de grado cargados en el sistema.
      
      Tus responsabilidades incluyen:
      - Guiar en los criterios de formato, normas de citación (APA) y metadatos requeridos.
      - Responder preguntas sobre estados de proyectos (Pendiente, Aprobado, Rechazado).
      - Ayudar en la búsqueda de temáticas o sugerir palabras clave para la catalogación del proyecto.
      
      Restricciones: 
      - Mantén un tono formal, profesional, institucional y extremadamente organizado.
      - No des información confidencial de usuarios ni modifiques estados directamente (indica que deben usar la interfaz del sistema para ello).
    `,
  });

  return model.startChat({ history: [], generationConfig });
};

// ==========================================
// 2. CHATBOT PARA DOCENTES Y COORDINADORES
// ==========================================
export const iniciarChatDocenteCoordinador = () => {
  const model = genAI.getGenerativeModel({
    model: "gemini-1.5-flash",
    systemInstruction: `
      Actúas como un asesor de gestión académica y metodológica para Docentes y Coordinadores del proceso de proyectos de grado.
      
      Tus responsabilidades incluyen:
      - Asistir en la evaluación de la coherencia de propuestas, objetivos y planteamientos del problema enviados por los estudiantes.
      - Dar recomendaciones basadas en metodologías de investigación científica.
      - Ayudar a redactar correcciones o retroalimentaciones constructivas para las entregas de los estudiantes.
      - Resolver dudas sobre el flujo de asignación de jurados o procesos internos de coordinación de la plataforma.
      
      Restricciones:
      - Tu tono debe ser académico, analítico, respetuoso y de apoyo técnico-pedagógico.
      - Nunca apruebes o repruebes un proyecto tú mismo; tu labor es sugerir mejoras al docente para que él tome la decisión.
    `,
  });

  return model.startChat({ history: [], generationConfig });
};

// ==========================================
// 3. CHATBOT PARA ESTUDIANTES
// ==========================================
export const iniciarChatEstudiante = () => {
  const model = genAI.getGenerativeModel({
    model: "gemini-1.5-flash",
    systemInstruction: `
      Actúas como un tutor virtual y metodológico amigable para Estudiantes que están desarrollando su proyecto de grado.
      
      Tus responsabilidades incluyen:
      - Ayudar a estructurar el proyecto (cómo redactar la justificación, objetivos específicos, marco teórico, etc.).
      - Sugerir metodologías de desarrollo, herramientas de software o arquitecturas según el tipo de proyecto.
      - Explicar las observaciones o correcciones que dejaron sus docentes de manera sencilla.
      - Guiar sobre los pasos que deben seguir en la plataforma para subir sus entregas correctamente.
      
      Restricciones:
      - Tu tono debe ser empático, motivador, claro y paciente.
      - ¡CRÍTICO!: No hagas el trabajo por el estudiante. No redactes el código completo del proyecto de grado ni escribas capítulos enteros. Guíalos con ejemplos, preguntas guía o plantillas conceptuales para que ellos aprendan a hacerlo.
    `,
  });

  return model.startChat({ history: [], generationConfig });
};

```

## 2. Cómo estructurarlo en tus carpetas del Backend

Para que encaje perfectamente con las rutas que tienes pensadas, puedes mapear los archivos en `backend/src/modules/AIClassificationModule/ia/` de la siguiente forma:

### Opción Limpia: Un archivo por módulo

Puedes crear un archivo por carpeta que importe el inicializador y exponga el endpoint del chat:

* **`backend/src/modules/AIClassificationModule/ia/biblioteca/chat.js`**
```javascript
import { iniciarChatBiblioteca } from "../geminiConfig.js";

const chatSession = iniciarChatBiblioteca();

export const conversarBiblioteca = async (req, res) => {
  const { mensaje } = req.body;
  try {
    const result = await chatSession.sendMessage(mensaje);
    res.json({ respuesta: result.response.text() });
  } catch (error) {
    res.status(500).json({ error: "Error en el chat de biblioteca" });
  }
};

```



*(Harías exactamente lo mismo para la carpeta de `Docente y coordinador/` y `estudiantes/` llamando a su función respectiva).*

## 3. Conexión hacia Angular (Frontend)

En tu frontend (`frontend/src/app/services/`), crearás un servicio de Angular común (por ejemplo, `chat-ia.service.ts`) para consumir estos endpoints pasando el mensaje del usuario y el rol, o apuntando directamente a la ruta correspondiente:

```typescript
import { Injectable } from '@angular/core';
import { HttpClient } from '@angular/common/http';
import { Observable } from 'rxjs';

@Injectable({
  providedIn: 'root'
})
export class ChatIaService {
  // Gracias a tu proxy.conf.json redirige automáticamente a localhost:5001
  private apiUrl = '/api/ai'; 

  constructor(private http: HttpClient) {}

  // Apunta al chatbot según el rol del panel donde se encuentre el usuario
  enviarMensaje(rol: 'estudiante' | 'docente' | 'biblioteca', mensaje: string): Observable<any> {
    return this.http.post(`${this.apiUrl}/chat/${rol}`, { mensaje });
  }
}

```

### Un par de consejos para tu arquitectura actual:

1. **El historial en memoria:** El método `.startChat()` que estamos usando mantiene el historial **mientras el servidor de Node esté corriendo y para un único flujo**. Si el usuario refresca la página de Angular o si varios usuarios entran a la vez al backend, el historial se cruzará o se borrará. Cuando implementes producción, te recomiendo recibir el arreglo del historial desde el Frontend (`history: req.body.historial`) y pasárselo a `startChat({ history })` en cada petición HTTP para que el chat sea verdaderamente multiusuario e independiente.
2. **Seguridad con JWT:** Como ya cuentas con la carpeta `middleware/` con protección por JWT y roles, no olvides inyectar estos middlewares en las rutas de estos chatbots para asegurar que un estudiante no consuma el endpoint del chatbot de docentes/coordinadores.
